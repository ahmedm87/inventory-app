Sync Algorithm (11 Steps)

1. Concurrency check (in-memory + DB guard)
2. Create SyncRun(RUNNING)
3. Enumerate warehouses (GetStorageList API → env fallback)
4. Fetch Fulfillmen inventory (paginated, 20/page, aggregate by SKU)
5. Fetch Shopify variants (GraphQL pagination, filter tracked + stocked)
6. Match SKUs (exact string match)
7. Compare quantities → build update batch (skip unchanged)
8. Batch inventorySetQuantities (100/mutation, rate limit backoff)
9. Record unmatched SKUs
10. Finalize SyncRun(COMPLETED) with stats
11. On exception → SyncRun(FAILED) with error message

Issues Discovered & Fixed During Testing

Fulfillmen API returns string Code, not number
Code: "100" not Code: 100 — fixed all comparisons to use String(body.Code)
Fulfillmen warehouse field is ShortName, not StorageCode
API docs implied StorageCode but actual response uses ShortName
Fulfillmen Code 105 = empty warehouse
Not in docs — means "no inventory info". Added graceful handling (return empty data)
Shopify API version deprecated
2025-01 no longer supported — updated to 2025-07
inventorySetQuantities requires ignoreCompareQuantity
Newer Shopify API versions require ignoreCompareQuantity: true or per-item compareQuantity
Variants not stocked at location
Some variants have no inventory level at the target location — skip them instead of trying to set quantity
Vite SSR module boundary
Config loaded in server.ts wasn't visible to Remix routes — moved to globalThis
Bearer token timing attack
Replaced !== comparison with crypto.timingSafeEqual on the trigger endpoint
